# 2026-04-15 Handoff

## 本次工作目标

本轮工作的目标是把当前项目的 registry 驱动链路收敛到新的规则：

- endpoint 不再由单个固定字段直接提供
- endpoint 统一由 `model_name` + `endpoint_category` 组装
- 渠道通过统一下拉影响 `model_name` 后缀
- 默认多媒体多输入口自动补 inputcount 控件
- registry 去掉动作逻辑字段
- registry 去掉模型覆盖和混合校验逻辑字段
- registry 字段命名尽量对齐 `ComfyUI_RH_OpenAPI`

## 已完成的改动

### 1. endpoint 组装链路完成

当前 endpoint 只由下面三部分决定：

- `model_name`
- `endpoint_category`
- 可选 `channel`

组装规则：

- `model_name` 作为前半段
- `endpoint_category` 转小写，空格转 `-`
- 如果 `channel` 非空，则直接把规范化后的 `channel` 追加到 `model_name` 后面

示例：

```text
model_name = nano-banana-pro
channel = Official API
endpoint_category = Image To Image
```

最终 endpoint：

```text
nano-banana-pro-official-api/image-to-image
```

### 2. 自动 inputcount 已接通

对于媒体多输入参数，只要满足：

- `type` 是 `IMAGE` / `VIDEO` / `AUDIO`
- `multipleInputs = true`
- `name` 是 `image` / `images` / `video` / `videos` / `audio` / `audios`

系统会自动补出：

- `image_inputcount`
- `video_inputcount`
- `audio_inputcount`

最大数量由媒体参数自身的 `maxInputNum` 决定。

这意味着默认场景不再需要在 registry 里显式写：

- `image_inputcount`
- `video_inputcount`
- `audio_inputcount`
- `inputcountParam`

前端动态输入 JS 本轮没有改，因为现有逻辑已经能识别后端自动生成的 count widget。

相关代码：

- [`nodes/node_factory.py`](/Users/huhuhu/Desktop/refactor/bizytrd/nodes/node_factory.py)
- [`web/js/dynamic_inputs.js`](/Users/huhuhu/Desktop/refactor/bizytrd/web/js/dynamic_inputs.js)

### 3. `transform` 已移除，改为有限 hook

registry 不再使用：

- `transform`
- `transform_*`

当前替代方案是有限 `valueHook`。这套机制后续已经继续演进为脚本路径 hook：

- `common.json_loads`
- `wan.custom_size`
- `wan.bbox_list`
- `wan.color_palette`

如果以后需要新增特殊行为，应优先在 [`core/hooks`](/Users/huhuhu/Desktop/refactor/bizytrd/core/hooks) 下增加命名 hook，再在 registry 中引用，不要重新引入自由动作逻辑。

### 4. 旧逻辑字段已移除

下面这些字段已经从主 registry 和主运行链路中移除：

- `api_node`
- `request_model`
- `request_model_from`
- `require_any_of`
- `require_any_message`

当前请求模型只由 `model_name` 决定，不再允许另一套覆盖来源。

### 5. 混合节点已进一步扁平化

原来 `Wan27Image` 的混合语义已拆成更扁平的节点定义：

- `BizyTRD_Wan27ImageProTextToImage`
- `BizyTRD_Wan27ImageTextToImage`
- `BizyTRD_Wan27ImageProImageToImage`
- `BizyTRD_Wan27ImageImageToImage`

这样就不再需要模型覆盖和“至少填一个”的混合校验字段。

### 6. 字段命名已统一向 RH 风格靠拢

当前 registry 主字段命名约定：

- 模型级保留 snake_case：
  - `internal_name`
  - `class_name`
  - `display_name`
  - `model_name`
  - `endpoint_category`
  - `output_type`

- 参数级元数据统一为 camelCase：
  - `fieldKey`
  - `defaultValue`
  - `multipleInputs`
  - `maxInputNum`
  - `flattenBatches`
  - `mediaItemType`
  - `sendIf`
  - `skipValues`
  - `onlyIfTrueParam`
  - `onlyIfFalseParam`
  - `onlyIfMediaAbsent`
  - `onlyIfMediaPresent`
  - `valueHook`

说明：

- 这套分层命名是折中方案
- 模型级字段暂未整体切成 RH 的 `name_en` / `name_cn` / `endpoint` 风格
- 但参数级已经尽量统一到 RH 式 camelCase，避免一会儿 snake_case 一会儿 camelCase
