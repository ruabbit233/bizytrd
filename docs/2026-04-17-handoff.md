### 8. 媒体 payload 规则已进一步收敛

当前前端节点层对媒体参数的规则已经明确为：

- 只要是媒体字段，写进 payload 时一律是 URL list
- 即使只有一个媒体输入，也发送单元素 list
- 只要 `maxInputNum > 1`，运行时就把它视为多输入媒体；不再需要单独的 `multipleInputs`
- 不再支持 `multipleInputs`
- 不再支持 `flattenBatches`
- 不再支持 `forceList`
- 不再支持 `mediaItemType`

这意味着前端节点层不再负责：

- 展开单个批量图片输入
- 判断单张媒体是发单值还是数组
- 把多个媒体字段拼成 typed media object array

如果后端需要把这些语义不同的媒体 list 再转换成某种 template 结构，应由后端 template 负责。

## 当前主 registry 的关键规则

当前主 [`models_registry.json`](/Users/huhuhu/Desktop/refactor/bizytrd/models_registry.json) 有这些明确约定：

- endpoint 由 `model_name + endpoint_category + 可选 channel` 组装
- `channel` 是唯一固定渠道参数
- `valueHook` 必须是脚本路径
- 不再使用 `hook*Param`
- 多输入媒体只由 `maxInputNum` 决定
- 媒体字段一律输出 URL list
- 不再使用 `multipleInputs` / `flattenBatches` / `forceList` / `mediaItemType`
- 输入显示顺序由运行时稳定排序决定
- 多媒体多输入场景优先依赖 `maxInputNum` 与自动 `inputcount`
